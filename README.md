- Consultas y dónde se guardan
Las consultas no se almacenan en la base de datos; solo se ejecutan cuando el usuario las pide. Lo que sí se puede guardar son vistas o procedimientos que contienen consultas ya preparadas.

- Rendimiento con índices
Los índices hacen que las búsquedas sean mucho más rápidas porque evitan revisar toda la tabla. Sin ellos, las consultas en tablas grandes se vuelven lentas y pesadas.

- Utilidad de vistas
Las vistas ayudan a simplificar el trabajo porque permiten tener consultas ya listas y mostrar solo la información necesaria. Facilitan reportes y reducen errores.

- Importancia de transacciones
Las transacciones garantizan que los cambios se hagan completos y correctos. Si algo sale mal, permiten regresar atrás para evitar daños en los datos.

- Uso de procedimientos
Los procedimientos sirven para automatizar acciones repetidas y mantener reglas claras dentro de la base de datos. Ahorran tiempo, reducen errores y hacen el sistema más ordenado.
